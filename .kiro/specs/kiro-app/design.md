# Design: Kiro App

## Architecture

Single-page web application using Node.js + TypeScript + Express on the backend, with a plain HTML/CSS/JS frontend served as a static file.

```
kiro-playwright-demo/
├── src/
│   └── server.ts        # Express server entry point
├── public/
│   └── index.html       # Single-page UI (HTML + inline CSS + inline JS)
├── dist/                # Compiled JS output (generated by tsc)
├── package.json
├── tsconfig.json
└── .kiro/
    └── specs/kiro-app/
```

## Components

### `src/server.ts`
- Creates an Express app
- Mounts `express.static` middleware pointing at `public/`
- Listens on port 3000
- Logs a startup message to the console

### `public/index.html`
Single self-contained HTML file with:
- A `<h1>` header: "Welcome to Kiro App"
- A `<input type="text">` field with placeholder "Enter your name"
- A `<button>` labelled "Submit"
- A `<p>` element (initially empty) that displays the greeting
- Inline `<script>` that reads the input value on button click and writes `Hello, [Name]!` into the greeting element

## Data Flow

```
User types name → clicks Submit
  → JS reads input.value
  → if non-empty: sets greeting element text to "Hello, [Name]!"
  → if empty: clears greeting element text
```

No server round-trip is needed for the greeting — it is handled entirely client-side by the inline script.

## TypeScript Configuration (`tsconfig.json`)
- `target`: ES2020
- `module`: CommonJS
- `outDir`: `./dist`
- `rootDir`: `./src`
- `strict`: true
